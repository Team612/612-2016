16 /**
17  * A {@link CommandGroup} is a list of commands which are executed in sequence.
18  *
19  * <p>Commands in a {@link CommandGroup} are added using the {@link
20  * CommandGroup#AddSequential(Command) AddSequential(...)} method
21  * and are called sequentially.
22  * {@link CommandGroup CommandGroups} are themselves {@link Command Commands}
23  * and can be given to other {@link CommandGroup CommandGroups}.</p>
24  *
25  * <p>{@link CommandGroup CommandGroups} will carry all of the requirements of
26  * their {@link Command subcommands}. Additional
27  * requirements can be specified by calling {@link
28  *CommandGroup#Requires(Subsystem) Requires(...)}
29  * normally in the constructor.</P>
30  *
31  * <p>CommandGroups can also execute commands in parallel, simply by adding them
32  * using {@link CommandGroup#AddParallel(Command) AddParallel(...)}.</p>
33  *
34  * @see Command
35  * @see Subsystem
36  */
